# CARD_STYLE — 스킬 카드 아트 통일 스타일 가이드

## 왜 통일하는가 (덱 일관성)

Loadout의 스킬·에이전트·MCP 카드는 수천 장이 한 컬렉션을 이룬다.
아트 스타일이 카드마다 달라지면 UI에서 한 덱처럼 보이지 않고,
희귀도 계층(전설→커먼)의 시각적 위계도 무너진다.
`CARD_STYLE` 상수를 모든 `buildPrompt("card", ...)` 호출의 prefix로 고정함으로써:

- 어떤 엔진(grok/chatgpt)으로 생성해도 동일한 레이아웃·배경·프레임 구조를 유지한다.
- 등급 차이는 프레임 액센트 색 하나로만 표현해 일관성과 구분성을 동시에 확보한다.
- 앱이 텍스트/스탯을 얹을 자리(중앙 아트 영역)를 간섭하지 않는다.

---

## 고정 스타일 규칙

| 요소 | 규칙 |
|---|---|
| 포맷 | 세로 트레이딩 카드 1장, 비율 **16:22** |
| 배경 | 다크 틸트(teal)/에메랄드 방사형 그라데이션 + 미세 광휘(sparkle) 입자 오버레이 |
| 프레임 베이스 | 두꺼운 골드 테두리 + 모서리 룬 장식 |
| 중앙 일러스트 | 해당 도메인 상징 반신(半身) 판타지 캐릭터, 림라이트, 하스스톤풍 유화 질감 |
| 텍스트 | **금지** — 이름·스탯·HUD 라벨 일절 없음 |
| 워터마크 | **금지** |
| 구도 | 단일 카드, 균일한 구도, 사진 아님 |

## 프롬프트 구조 (buildPrompt 출력 예)

```
[CARD_STYLE prefix]
  세로 트레이딩 카드 1장, 비율 16:22.
  배경: 다크 틸트/에메랄드 방사형 그라데이션, 미세 광휘 입자 오버레이.
  프레임: 두꺼운 골드 베이스 테두리, 모서리 룬 장식, 등급별 액센트 색(...).
  중앙: 해당 스킬 도메인 반신 일러스트, 림라이트, 하스스톤풍 유화 질감.
  글자·텍스트 일절 없음. 워터마크 금지. 단일 카드.
[rarity frame]
  에픽 등급 카드: 딥퍼플(#7B2FBE) 액센트 프레임, 마법진 빛.
[subject + motif]
  "my-skill"(skill, security)를 상징하는 방패와 룬 문자를 든 수호자.
  배경 단색 또는 투명.
```

## 변경 금지 사항

- `CARD_STYLE`에서 "글자·텍스트 없음" 조항 제거 금지
- 비율 16:22 변경 금지 (앱 카드 레이아웃과 연동)
- 프레임 베이스(골드) 변경 금지 (액센트 색만 등급별로 변주)
