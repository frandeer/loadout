# 09 · 리서치 — 뽀짝이의 OpenClaw 클래스 (2026-06-13)

> 출처: https://bbojjak-viewer.vercel.app/lessons — 23개 레슨 전수 조사.
> GPTers 커뮤니티 AI 스터디에 실전 투입된 에이전트 '뽀짝이'(OpenClaw 위 Claude 자율 운영 2개월 기록).
> 5개 섹션: 에이전트의 집 / 자율 운영 / 팀·세계 확장 / 보안 / 장기 생존.
> 특징: 이론이 아니라 "사고 → 원인 분석 → 규칙화" 인시던트 기반 기록.

## 핵심 유용 정보 톱 10

1. **읽기가 토큰의 주범 + RTK** (L16): `git status` 출력의 70~80%는 노이즈. `rtk` 프리픽스로 78% 압축. 자동화 실패 시 규칙, 규칙 실패 시 문서.
2. **스킬 발견은 description이 전부** (L10): SKILL.md frontmatter `description`이 매칭의 전부. 로딩 우선순위: 워크스페이스 > 공유(`~/.claude/skills/`) > 번들.
3. **사고 → 구체적 규칙 화석화** (L5, L22): 추상 규칙("불필요한 반복 자제")은 실패, 실행 가능 규칙("KST 8:30~9:30에만 발송")만 작동. AGENTS.md는 "실수의 화석층".
4. **3계층 자동화론** (L11): API > CLI > GUI 순 안정성. GUI 자동화는 검증·폴백으로 감싸 CLI처럼. 전송 후 독립 경로로 2차 검증.
5. **서브에이전트 위임 원칙** (L12): 판단 제로 작업만 위임. 컨텍스트는 task 문자열에 전부 명시. GUI 점유 작업은 위임 금지.
6. **Heartbeat vs Cron 분리** (L9): 주기 순찰=Heartbeat, 정시 산출물=Cron(독립 세션·저가 모델). 양쪽에 같은 작업 두면 중복 실행.
7. **신뢰도 기반 라우팅 + 비-LLM 가드레일** (L23): 확신 높은 것만 자동응답. LLM 앞단 비-LLM 필터(노이즈 제거·디바운싱)로 AI 호출 80% 절감. 안전망 크론은 LLM 없이 bash/curl.
8. **모델 페일오버 2단계** (L19, L21): auth 프로필 로테이션 → 모델 캐스케이드. 지수 백오프 쿨다운, 세션 고정으로 캐싱 유지. 개인/에이전트 계정 분리.
9. **보안: 에이전트 분리 > 스킬** (L17, L18, L20): 외부 노출 에이전트는 read-only·allowlist·최소 정보로 분리. 인젝션 패턴은 `learnings/`에 자동 누적.
10. **HTML→Playwright 이미지 생산** (L13): 텍스트 정확도 필요한 카드는 AI 생성 대신 HTML 템플릿 + `deviceScaleFactor:2` 스크린샷. 18장 4.5시간 → 2분.

## Loadout 접목 아이디어 (백로그 후보)

- **description 품질 스탯**: `scan.mjs` `computeStats()`에 description 유무·트리거 조건 구체성을 반영 — 발견성(장착 가치)을 스탯으로. (L10/L18)
- **보안 스캔 스탯**: 외부 네트워크 호출 여부(오프라인 가산점), 스크립트 동봉, exec 의존도를 "안정성/위험" 스탯으로. AI judge 프롬프트에 기준 추가. (L18)
- **덱 비대화 경고**: 장착 수 ↑ → 시스템 프롬프트 비대 → 중요 규칙 묻힘. 팀 화면에 토큰 비용/과적 경고 — 마스터 플랜 Phase 3 "코스트=마나"와 동일 방향. (L18)
- **AI judge 쿨다운**: 실패 엔진 지수 백오프(현재는 1회 프로브 캐시만). (L19)
- **카드 아트 HTML 파이프라인**: 텍스트 중요한 카드 프레임은 HTML 템플릿 + Playwright 스크린샷이 CDP+ChatGPT보다 일관·정확·빠름 — `gen-cards.mjs` 보완 엔진 후보. (L13)
- **카드 learnings = 경험치**: verify 결과·사용 피드백을 카드별 누적 기록 → 재평가 반영. Phase 3 "사용량=경험치"와 합류. (L22)
