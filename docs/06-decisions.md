# 06 · 의사결정 로그

`grill-me` 인터뷰로 확정한 결정들. 각 결정은 되돌릴 수 있지만, 바꾸려면 이 문서를 갱신한다.

| # | 주제 | 결정 | 근거 / 대안 |
|---|------|------|------------|
| 1 | **이름** | **Loadout** | 게임 "장비 세트" — 카탈로그에서 골라 장착하는 컨셉과 일치. (대안: Quartermaster, Arsenal, Grimoire) |
| 2 | **위치** | `D:/lab/loadout` | `ref/`(지저분한 원본 창고)와 **분리된 깨끗한 별도 프로젝트**. |
| 3 | **빌드 방식** | 스캔 스크립트 + 정적 SPA + **얇은 로컬 서버** | 탐색/비교는 정적, clone·장착·업데이트 "행동"만 서버. DB 없음. (대안: 순수 정적 / 풀 웹앱+SQLite) |
| 4 | **적용(Equip)** | `~/.claude`에 **심링크 설치** | "장착=즉시 사용"을 가장 단순히. Win은 dir junction/복사 fallback. (대안: 플러그인 마켓플레이스 / staging만) |
| 5 | **소스 전략** | 전부 `loadout/sources/`에 **독립 clone** | `ref`와 완전 분리해 깔끔하게 소유. `sources/<owner>__<repo>`로 이름 충돌 방지. (대안: ref 읽기전용 스캔) |
| 6 | **능력치** | **자동 기본 스탯** + 요청 시 **AI `+99`** + 등급 | 평소 무료·즉시, 깊게 볼 때만 AI 비용. (대안: 전부 자동 / 전부 AI / 수동) |
| 7 | **MCP 소스** | repo 자동탐지 + **내 `~/.claude` 설정에서 가져오기** | repo엔 mcp가 드묾 → 실제 쓰는 MCP가 중심. (대안: 큐레이션 레지스트리 포함 / 제외) |
| 8 | **기술 스택** | **바닐라 JS + 최소 의존성** | 기존 `cheatsheet.html` 성공 사례. 빌드 없음. (대안: Vite+React / 기존 파일 확장) |
| 9 | **첫 범위** | **MVP부터 단계적** | Phase 1(카탈로그)→2(장착/업뎃)→3(AI/연출/이미지). (대안: 한 번에 완성) |
| 10 | **이미지** | 기존 14개 재사용 + 필요 시 **codex-image 생성** | 100~1000개 사전 생성은 과함. (대안: 전부 사전생성 / 없음) |

## UI 메타포 결정 (대화로 구체화)

| 실제 | 게임 개념 | 핵심 |
|------|----------|------|
| Skill | 🃏 카드 (포켓몬/하스스톤) | 앞: 요약, **클릭 시 3D 플립** → 뒤: 상세 |
| Agent | 🧑 팀원/유닛 | 로스터 → **포메이션 배치**(축구·턴제), 팀 프리셋 |
| MCP | ⚔️ 무기/장비 | 장비 슬롯, 능력 부여 |
| 장착 | 🎒 인벤토리/로드아웃 | 장착=인벤토리행 + 실제 활성화 |
| 비교 | ⚔️ 스탯 대결 | 막대 비교 + AI WINNER 판정 |
| 등급 | Common→**Legendary(foil)** | 종합 점수 기반 카드 프레임 |

## 열린 질문 (구현 중 결정)

- 시드 소스: `ref/` 기존 clone을 복사 시드 vs 전부 새 clone. (로드맵 권장: 큰 repo는 복사 시드)
- AI judge 기본값: `claude -p` 헤드리스로 충분한지, 토큰/속도.
- 포메이션 → 실제 OMC `/team` export 형식.
- 등급 점수 가중치 튜닝(실데이터 보고 조정).
