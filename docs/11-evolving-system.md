## 0. 성격 전환 — 무엇이 되려 하는가

지금까지 Loadout은 **수동 카탈로그**였다. 다음 단계의 목표는 **컨텍스트 거버넌스 콘솔**이고,
최종 지향은 **자기 진화 시스템**이다.

```
   현재 (수동 카탈로그)          →   목표 (능동 거버넌스)          →   지향 (자기 진화)
   ─────────────────               ──────────────────              ───────────────
   이름기반 그룹                     의미기반 온톨로지 그래프            관찰→제안 루프 자동화
   읽기전용 ~/.claude                vault 중앙화 + 링크 on/off         "이건 꺼라/이건 켜라" 능동 제안
   uses 데이터만 보유                 미사용/모름 능동 가시화             사용 패턴으로 태그·관련도 자가 보정
   regex 위험 3종                    SkillSpector 16종 + 위험점수        신규/변경 자산 자동 위험 검사
   memory만 프로젝트 인식             skill·mcp도 프로젝트별 facet        프로젝트 맥락에 맞는 로드아웃 추천
```

**핵심 명제(08 마스터플랜 계승):** 병목은 모델이 아니라 환경 품질. "장착 수 = 컨텍스트 비용."
→ 이 시스템의 일은 **절제를 돕는 것**: 안 쓰는 걸 끄고, 쓸 것을 의미기반으로 찾아주고, 위험한 걸 막는다.

## 1. 자기 진화 루프 — "성장"의 구체적 정의

막연한 "자기 진화"가 아니라, 4개의 닫힌 루프로 정의한다. 각 루프는 **이미 있는 씨앗**을 가진다.

| 루프 | 입력(관찰) | 출력(행동) | 현재 씨앗 |
|---|---|---|---|
| **관찰 루프** | 세션 로그의 실제 호출(`usage.mjs`) | "이건 0회 → 꺼라", "이건 자주 씀 → 항상 켜라" 제안 | `uses`/LV 집계 O |
| **흡수 루프** | 새 repo/스킬 ingest | 임베딩·온톨로지 추출 → 그래프에 자동 편입 | `scan` + `clone` O |
| **보호 루프** | 자산 콘텐츠 변경 | 위험 재검사 → 점수/배지 갱신, CRITICAL 차단 | regex risk O |
| **학습 루프** | 해결한 어려운 문제 | 패턴을 스킬 카드로 드랍 | `drop.mjs` O |

→ **자기 진화 = 이 4루프가 사람 개입 없이 "관찰→제안"까지 자동으로 돈다.** 적용(끄기/켜기/병합)은 사람이 확인(비-목표 유지).

## 2. 단계별 체크리스트 (점진 개선)

> 권장 순서: **A(빠른 win) → B(미사용 가시화) → C·D(의미 그래프=핵심) → E(vault) → F(프로젝트)**.
> 각 단계는 독립 배포 가능하게 쪼갠다. scan 멱등·서버 zero-dep·read-only 원칙 유지(§5).

### Phase A — 위험 감지 강화 (SkillSpector) 🟢 빠른 win·독립
NVIDIA SkillSpector(Apache-2.0, 16개 취약점 카테고리, CLI/Docker, JSON/SARIF, 0~100 점수).
AI judge 멀티엔진 셸아웃 패턴을 그대로 재사용 → optional dependency.

- [ ] `skillspector` 설치 감지(Python/Docker 프로브, 엔진 가용성 캐시 — `engines` 패턴 차용)
- [ ] `POST /api/risk-scan {id}` — `skillspector scan <path> --format json` execFile(타임아웃, 인자배열 인젝션 차단)
- [ ] 결과 파싱 → 카드에 `riskScore(0~100)` + `riskLevel(LOW/MED/HIGH/CRITICAL)` + 카테고리 배열
- [ ] 미설치 시 기존 regex 3종으로 폴백(+`note`)
- [ ] UI: 카드 위험 배지, CRITICAL은 장착 경고/차단, 인벤토리 "위험 일괄 검사"
- [ ] 결과 캐시(`data/risk.json`, 콘텐츠 해시 키 → 변경 시만 재검사 = 보호 루프)

### Phase B — 미사용/모름 가시화 (데이터는 이미 있음) 🟢 UI·로직만
`uses=0` = 설치됐지만 한번도 안 쓴 스킬. 데이터 존재, **노출만 없음**.

- [ ] 덱 필터: "미사용(0회)" / "거의 안 씀(하위 N%)" / "자주 씀"
- [ ] 인벤토리: 장착됐지만 미사용 → **"끄기 추천"** 섹션 (관찰 루프 1차 산출)
- [ ] 카드에 "마지막 사용" / "한번도 안 씀" 신호 표기
- [ ] 컨텍스트 절감 추정: "미사용 N개 끄면 ~X tk 절약" (마나 모델 재사용)

### Phase C — 의미 사이드카 (임베딩·관련도) ⭐ 핵심·미지수
스킬을 직접 수정하지 않고 **별도 사이드카**로 재가공(읽기전용 원칙 일치).

- [ ] `src/enrich.mjs`(신규, scan과 분리) — 각 카드 desc+본문 임베딩
- [ ] 저장소 결정 후 구현 → §4-D1 (후보: `node:sqlite` 단일파일 vs `embeddings.json`)
- [ ] 관련도 = 코사인 유사도, 임계치 이상 엣지 생성 → `data/graph.json`
- [ ] 증분 갱신(콘텐츠 해시 기준, 변경분만 재임베딩 = 비용 통제)
- [ ] `GET /api/graph` — 노드/엣지 제공
- [ ] 검증: "비슷한 스킬 3장" 추천이 이름기반 `group`보다 나은지 눈으로 확인

### Phase D — 온톨로지 그래프 UI ⭐ 핵심
"언제 써야 할지 모름"의 직접 해결.

- [ ] LLM으로 각 자산에서 `{capability, domain, requires(mcp), produces, when_to_use}` 추출(verify 인프라 재사용, 실패 시 휴리스틱)
- [ ] 온톨로지 스키마 확정 → §4-D2
- [ ] 노드 속성으로 그래프 영속(`data/ontology.json` 또는 graph에 병합)
- [ ] 그래프 뷰: 자산 중심 4~8노드 축약형(design.md `memory-graph`/`dependency-graph-mini` 토큰 재사용)
- [ ] "이 스킬은 이럴 때(when_to_use) 쓰면 좋다" 인스펙터 노출

### Phase E — 링크기반 vault (소유권 반전) ⭐ 진짜 on/off
현재 `~/.claude/skills`가 원본이라 끄려면 삭제해야 함. **중앙 vault → 켠 것만 junction**으로 뒤집는다.

- [ ] vault 위치 결정 → §4-D3
- [ ] **흡수(import) 마이그레이션**: 글로벌 상주 스킬을 vault로 이동 + 원위치에 junction (dry-run·백업 선행)
- [ ] 인벤토리 = 진실의 단일 출처: "상주(링크 존재)" vs "보관(vault만)" 구분
- [ ] on/off = 링크 생성/제거(원본 보존). 장착 junction 메커니즘 재사용(이미 검증)
- [ ] 안전장치: 다른 도구가 직접 참조하는 스킬 경고, 롤백 경로

### Phase F — 프로젝트별 facet
memory만 `scope=slug` 인식 → skill·mcp도 프로젝트 귀속.

- [ ] scan에 project facet: 워크스페이스 `<proj>/.claude/skills`·`<proj>/.mcp.json` 탐지 → `scope:"project:<name>"`
- [ ] UI "프로젝트별 보기" 축, 프로젝트 ↔ 글로벌 자산 비교
- [ ] (자기진화 연계) 프로젝트 맥락에 맞는 로드아웃 추천

### Cross-cutting — 자기 진화 자동화 (위 단계 위에 얹음)
- [ ] 관찰→제안 데일리 다이제스트: "미사용 N · 위험 M · 신규 K · 추천 켜기 J"
- [ ] 사용 패턴으로 태그/관련도 자가 보정(같이 쓰인 스킬 = 엣지 가중)
- [ ] 신규/변경 자산 자동 위험 검사(보호 루프 무인화)

## 5. 불변 원칙 (모든 단계가 지킨다)

- `scan.mjs` **멱등·read-only** 유지. 의미·위험·임베딩은 **별도 사이드카 단계**(enrich/risk)로, scan에 섞지 않는다.
- 서버 **zero-dep** 유지. 외부 도구(SkillSpector, 임베딩)는 **optional dependency + 셸아웃 + 폴백**.
- 데이터는 **JSON/단일파일 DB** (diff·백업·이식성). DB 서버 도입 금지.
- 파일 **이동/삭제는 dry-run·백업 선행**. 자산 원본 수정 금지(병합·드랍은 제안/신규 생성까지).
- ID는 path-namespaced 유지. 한국어 UI/주석 유지.
