# 03 · UI/UX 디자인 (Orbital Launch Monitor)

> 현재 우선 컨셉은 첨부 레퍼런스 이미지의 **gstack ▷ SHIP · MONITOR** 스타일을 **카드 안의 스킬 이미지 생성 방향**으로 사용하는 것이다. 전체 앱 레이아웃은 기존 Loadout 대시보드 구조를 유지하고, 카드 아트/이미지 슬롯에만 NASA mission-control, blueprint, 아주 약한 cyber/laser 감각을 섞는다.

## 카드 이미지 생성 방향

- 스킬 이미지는 밝은 blueprint paper 위에 로켓 궤적, 위성 원격측정, 얇은 레이저 라인, 데이터 궤도선을 사용한다.
- 이전 디자인의 미래형/사이버틱 감각은 **아주 약하게**만 섞는다. 예: 얇은 cyan laser trace, subtle glow, precise HUD ticks.
- Palantir/국방부처럼 무겁거나 군사적인 톤은 피하고, NASA/JPL 엔지니어링 다이어그램처럼 진지하지만 낙관적으로 보이게 한다.
- 카드 바깥 전체 레이아웃은 레퍼런스 이미지처럼 SHIP/MONITOR 2단 패널로 바꾸지 않는다.

## 시각 규칙

- 카드 이미지 슬롯은 `#F7FBFF` 계열 밝은 종이색과 얇은 blueprint grid를 사용한다.
- 이미지 안의 주요 라인은 `#2E73DF` launch blue, `#22965A` orbit green, 약한 `#38D6C6` laser cyan을 사용한다.
- 과한 3D, 어두운 금속, amber military glow는 사용하지 않는다.

## 카피 톤

- `작전 자산`은 유지 가능하지만, 화면 큰 흐름은 `SHIP`, `MONITOR`, `발사 준비`, `궤도 감시`, `원격 측정`, `체크리스트`를 우선한다.
- `신호 감정`은 `상태 점검` 또는 `원격 측정 점검`으로 해석 가능하다.
- `작전 투입`은 `임무 배치` 또는 `발사 큐 연결`로 해석 가능하다.

# Legacy · UI/UX 디자인 (게임풍)

> 컨셉: **데스티니·에이펙스식 '수집 → 로드아웃'** 감성 + 트레이딩 카드 UI. "복잡하지만 재밌게, 물 흐르듯". 다크 테마 베이스(기존 cheatsheet 팔레트 계승).

## 화면 맵

```
┌──────────────────────────────────────────────┐
│  HEADER: 🎒 Loadout   [검색] [필터] [+ Link 추가] │
├──────────────────────────────────────────────┤
│  탭:  🃏 컬렉션 │ 🎒 인벤토리 │ 🧑 포메이션 │ ⚔️ 무기고  │
├──────────────────────────────────────────────┤
│                                              │
│   (탭별 콘텐츠)                                  │
│                                              │
└──────────────────────────────────────────────┘
```

4개 탭 = 4개 게임 모드:
1. **🃏 컬렉션** — 전체 skill 카드 둘러보기/비교/장착
2. **🎒 인벤토리** — 내가 장착한 것(= 실제 활성 세트)
3. **🧑 포메이션** — agent 팀 배치 보드
4. **⚔️ 무기고** — mcp 장비 슬롯

## 🃏 컬렉션 — 스킬 카드

### 카드 앞면 (트레이딩 카드 스타일)

```
        ╔═══════════════════════╗   ← 등급 프레임(색/foil)
        ║ ⚡7        🃏  pdf   ⭐ ║   ← 좌:파워(마나) 우상:등급보석
        ║ ┌───────────────────┐ ║
        ║ │                   │ ║
        ║ │   [카드 아트/이미지]  │ ║   ← 매핑 이미지 or 카테고리 일러스트
        ║ │                   │ ║
        ║ └───────────────────┘ ║
        ║  PDF · Document        ║   ← 이름 · 카테고리
        ║  "추출·병합·분할 처리"     ║   ← 한 줄 설명
        ║ ⭐84 ⚡71 🔄90 🎯88    ║   ← 미니 스탯 바
        ║ ────────── [anthropics]║   ← 소스 배지
        ╚═══════════════════════╝
```

- **등급별 프레임**: Common 회색 / Uncommon 초록 / Rare 파랑보석 / Epic 보라 / **Legendary 주황 + 홀로그래픽 foil**(CSS conic-gradient shimmer 애니메이션).
- **호버**: 카드가 살짝 떠오르며 3D 틸트(마우스 추적 parallax) + 글로우. 트레이딩 카드 호버 감성.
- **🆕 배지**: 업스트림 변경 시 카드 모서리에 리본.
- **장착됨**: 장착한 카드는 ✅ + 살짝 어두워지거나 골드 테두리.

### 카드 뒷면 (클릭 → 3D 플립)

클릭하면 Y축으로 **카드가 뒤집혀**(CSS `transform: rotateY`) 상세를 보여준다.

```
        ╔═══════════════════════╗
        ║ pdf            [✕ 닫기] ║
        ║ ─────────────────────  ║
        ║ 📖 언제 쓰나             ║
        ║  PDF 추출/병합/분할/폼... ║
        ║ 🛠️ 도구: Read, Bash     ║
        ║ 📊 스탯 (풀)            ║
        ║   인기   ████████░ 84   ║
        ║   파워   ███████░░ 71   ║
        ║   신선도 █████████ 90   ║
        ║   AI유용성 ░░░ (검증전)   ║
        ║ 🔗 anthropics/skills    ║
        ║ ─────────────────────  ║
        ║ [⚔️ 비교] [🤖 검증+99]    ║
        ║ [🎨 이미지생성] [🎒 장착]  ║
        ╚═══════════════════════╝
```

- **장착(Equip)** 버튼 → 카드가 인벤토리로 **날아가는** 트랜지션(물 흐르듯) → `~/.claude` 연결.
- **검증(+99)** → AI judge 호출, 채점되면 스탯 바에 `+99`가 차오르고 등급 상승 가능(파티클 연출).
- **비교** → 같은 그룹 후보와 스탯 대결 화면(아래).
- **이미지생성** → codex-image로 카드 아트 생성 → 즉시 카드에 반영.

### 컬렉션 인터랙션

- **검색/필터**: 종류·등급·카테고리·소스·"장착가능/장착됨"·"🆕 업데이트있음".
- **정렬**: 종합점수 / 각 스탯 / 신선도 / 인기.
- **팩 오프닝 연출**: 새 repo를 link로 추가하면 추출된 카드들이 "팩에서 쏟아지듯" 등장.
- **가상 스크롤**: 100~1000장도 부드럽게(렌더링은 보이는 카드만).

## ⚔️ 비교 (스탯 대결)

중복 그룹의 후보들을 나란히. 데스티니 장비 스탯 비교처럼.

```
   pdf (anthropics)        vs        pdf-tools (other)
   ⭐84 ████████░                     ⭐41 ████░░░░░
   ⚡71 ███████░░                     ⚡66 ██████░░░
   🎯88 ████████░                     🎯52 █████░░░░
   ──────────────                     ──────────────
   🤖 AI 판정:  ← [POST /verify]
   "anthropics 판이 +18 우세 — 더 완성도 높고 폼 처리 지원.
    other는 가볍지만 폼 미지원. → anthropics 장착 권장."
```

- 우세한 쪽에 **WINNER 왕관**.
- 결과는 `verdicts.json`에 캐시 → 다시 안 물어봄.
- 병합 제안 시 "둘 다 보관 / 우세한 것만 장착" 선택(자동 삭제는 안 함).

## 🎒 인벤토리

장착한 카드/유닛/무기가 모이는 곳 = **실제 활성화된 세트**.

- 덱처럼 정리. 카드 제거 시 `~/.claude` 연결도 해제(`/unequip`).
- "이 세트가 지금 Claude Code에서 활성" 상태를 명확히 표시.
- 슬롯/카테고리별 그룹(생산성/디자인/디버깅 등).

## 🧑 포메이션 — 에이전트 팀 보드

> agent = 부하직원/유닛. 축구처럼 **포지션에 배치**해 팀을 구성.

```
   로스터(벤치)              포메이션 보드
  ┌──────────┐        ┌─────────────────────┐
  │ executor │        │   🎯 planner         │  ← 공격형: 설계/기획
  │ planner  │  드래그 │                     │
  │ reviewer │  ───►  │ ⚡executor ⚡executor │  ← 미드필더: 실행
  │ explorer │        │                     │
  │ verifier │        │ 🛡️reviewer 🛡️verifier│  ← 수비형: 검증/리뷰
  └──────────┘        └─────────────────────┘
```

- 유닛 카드도 스탯/등급 보유(모델 tier = 등급 힌트: opus=Legendary 느낌).
- 포지션이 곧 **역할 파이프라인**(기획→실행→검증) — OMC `team` 파이프라인과 매핑 가능.
- 배치한 팀을 **프리셋으로 저장** → 나중에 그 팀 구성을 불러와 활용.
- (확장) 저장한 팀을 OMC `/team`이나 워크플로 설정으로 export.

## ⚔️ 무기고 — MCP 장비

> mcp = 무기/장비. 장착하면 능력(도구)이 캐릭터에 부여되는 느낌.

- 장비 슬롯 UI. 각 MCP = 부여하는 도구 목록을 "능력치 보너스"로 표시.
- 내 `~/.claude` 설정에 등록된 MCP는 "장착됨"으로, repo에서 발견된 건 "획득 가능"으로.
- 장착/해제 = CC 설정 반영(또는 안내).

## 비주얼 톤 & 접근성

- **팔레트**: 기존 cheatsheet 다크(`--bg:#0e1117`, 등급별 액센트색) 계승 → 일관성.
- **모션**: 플립/틸트/날아가기/파티클은 `prefers-reduced-motion` 존중(끌 수 있게).
- **성능**: 카드 아트는 lazy-load, 없는 이미지는 등급/카테고리 기반 placeholder.
- **이해 쉽게**: 어려운 용어엔 호버 툴팁(예: "MCP가 뭐야?" 1줄 설명). 카드 자체가 설명서가 되게.
