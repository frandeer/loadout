## 합의된 착지 의도

대시보드 1순위 의도 = **(A) "지금 내 Claude 상태"** — 무엇이 켜져 있고 / 컨텍스트를
얼마나 먹고 / 뭘 꺼야 하나. (발견·재미 B, 맥락 추천 C는 비채택.)

## 대시보드(관제탑) = 3초 스캔 다이제스트

1. **헤드라인 = 컨텍스트 토큰 무게**(개수 아님). 예: `상시 컨텍스트 ~3,200 tok (장착 1,800 +
   설치 베이스 1,400)`. **반드시 "추정" 표시** — 비용은 `바이트 ÷ 4` 근사(실제 토크나이저 아님,
   상한 캡). 툴팁: "바이트 기반 근사 — 실제 토큰과 다를 수 있음". 종류별 개수는 보조로 강등.
2. **정리 = 환수형만.** "켜진 무거운 자산 → 보관 시 `-N tok`"만 남긴다. **중복(dedup)은
   자산 탭으로 이동** — 중복 사본은 대개 로드돼 있지도 않아 꺼도 무게가 안 줄어든다(상태 아님,
   카탈로그 위생). 헬스 칩엔 "중복 그룹 N" 신호로만 잔류.
3. **사용 현황 패널 폐기.** 과거 이력 + 세션 로그 희소 → A와 무관·노이즈.
4. **헬스 칩 유지**(분기 등).
5. ~~"최근 변경(NEW)" 패널~~ — **백로그.** NEW는 현재 추적 자체가 없고(firstSeen 부재),
   scan.mjs에 넣으면 멱등성 불변식 위반. 구현 시 *서버*가 이전 스냅샷(contentHash 집합) 대비
   diff로 계산(scan 바깥). 별도 작업 + ADR.

## 무기고 지도 = 펼쳐 탐색하는 캔버스 (전 "그래프" 탭 대체)

- **형식 = 도메인 군집 트리맵.** force node-link 폐기(헤어볼·무의미 엣지). 근거 ADR 0001.
- **모집단 = 전체 카탈로그**(보관·미장착 포함). "무기고를 다 펼쳐 본다".
- **블록 = `category`**(현재 12개, 116개가 ~10개씩 고른 분포 — 묶음에 이상적).
- **면적 = 자산 개수**(무게 아님 — 무게로 하면 보관-only 도메인이 면적 0으로 증발).
  무게는 **블록 색/주석**으로 얹는다(`19개 · 켜짐 ~6k tok` + 켜진 무게 높을수록 진한 테두리).
- **타일 색 = 로드 상태**: ■장착 / ▨설치 베이스 / □보관.
- **관계 = 군집 멤버십**으로 표현. pairwise 동반(co-use)은 로그 희소로 *정직하게 안 그림*,
  태그 유사도 엣지는 폐기.
- **행동 가능(최소형):** 타일 클릭 → 상세 패널(기존), 호버 → 즉석 장착/해제/보관 토글 하나,
  다중 선택 → 일괄 작업(구 그래프에서 계승). dedup·세부는 자산 탭.

## 남은 잔가지(구현 시 기본값 — 다르면 수정)

- 지도 상단 컨트롤: **종류 필터(skill/agent/mcp/memory) + 검색(타일 하이라이트) + 상태
  필터(장착/설치/보관)** 만 계승. 구 그래프의 scope(ego/all)·관계 토글·force 관련 컨트롤은 제거.
- 트리맵 라이브러리: 기존 의존성 최소 원칙상 경량 직접 구현 또는 소형 라이브러리 검토.
