# 00 · 비전 & 게임 메타포

## 문제 (Why)

지금 Claude Code 환경에는 스킬·에이전트·MCP가 **수백~수천 개**나 깔려 있거나 참조용으로 clone돼 있다.
하지만 실제로는:

- 🤯 **뭐가 있는지 모른다** — `ref/`에만 `antigravity-awesome-skills`(≈1,377), `Claude-Code-Game-Studios`(48 agents/37 skills) 등 수천 개가 흩어져 있음.
- 🤔 **언제 써야 할지 모른다** — 설명이 길고 흩어져 있어 "이 스킬을 지금 써야 하나?" 판단이 어렵다.
- 👯 **중복이 많다** — 비슷한 스킬이 repo마다 존재. 어느 게 더 나은지 비교 기준이 없다.
- 🔄 **업데이트를 모른다** — `pull_all.sh`로 당겨와도 "무엇이 새로 바뀌었는지" 보이지 않는다.
- 😵 **이름이 겹친다** — repo들이 죄다 `skills/`, `skill` 이름을 써서 그냥 모으면 충돌한다.

→ **체계적으로 모으고, 비교하고, 골라 쓰는 단일 시스템**이 없다. 그게 Loadout이다.

## 해결 (What)

> 흩어진 skill/agent/mcp를 한곳에 모아 **게임 카드처럼** 펼쳐 보고,
> 능력치로 비교해 **좋은 것만 골라 장착**하면 바로 쓸 수 있는 개인 시스템.

핵심 가치는 세 가지:

1. **발견(Discover)** — 거대한 카탈로그를 재미있게 둘러본다 (포켓몬/하스스톤 카드).
2. **판단(Compare)** — `⚡+99` 스탯과 등급으로 "어떤 게 좋은지" 한눈에. 깊게는 AI 검증.
3. **장착(Equip)** — 고른 것을 `~/.claude`에 연결해 **즉시 사용 가능**하게.

## 게임 메타포 (How it feels)

전체 컨셉은 **"나만의 로드아웃을 꾸리는 게임"**. 실제 기술 객체를 게임 오브젝트로 1:1 매핑한다.

| 실제 | 게임 개념 | 비유 | 핵심 인터랙션 |
|------|----------|------|--------------|
| **Skill** | 🃏 스킬/아이템 카드 | 포켓몬·하스스톤 카드 | 컬렉션에서 구경 → **클릭 시 카드 뒤집힘**(앞: 요약 / 뒤: 상세) → 장착 |
| **Agent** | 🧑‍🤝‍🧑 팀원/유닛 | 축구 선수(메시 배치)·턴제 팀원 | 로스터에서 드래프트 → **포메이션 보드에 배치** → 팀 프리셋으로 저장 |
| **MCP** | ⚔️ 무기/장비 | 하스스톤 영웅 무기·장비 슬롯 | 장비 슬롯에 장착 → 능력(도구) 부여 |
| 장착 세트 | 🎒 인벤토리 / 로드아웃 | 내 가방·덱 | 장착한 카드/유닛/무기가 모이는 곳 = 실제 활성화된 세트 |
| 능력치 | ⚡ 스탯 `+99` · 등급 | 공격력/체력·레어리티 | 자동 기본 스탯 + 요청 시 AI가 `+99` 채점 |
| 비교 | ⚔️ 스탯 대결 | 카드 vs 카드 | 두 항목 스탯 막대 나란히 + AI "우세한 쪽" 판정 |
| 업데이트 | 🆕 NEW 배지 | 신규 카드 입수 | 업스트림 변경 감지 → 업데이트/유지 선택 |

### 한 줄 시나리오

> "GitHub에서 멋진 스킬 repo 링크를 붙여넣는다 → Loadout이 clone하고 카드로 펼쳐 보여준다 →
> 비슷한 카드 3장을 비교하니 AI가 *'B가 +12 우세'* 라고 알려준다 → B를 **장착** →
> 인벤토리에 들어가고 `~/.claude`에 연결돼 다음 세션부터 바로 쓴다."

## 비-목표 (Non-goals, 적어도 초기엔)

- ❌ 클라우드 호스팅/멀티유저 — **로컬 개인용**.
- ❌ 무거운 프레임워크/DB — 바닐라 JS + JSON 인덱스로 가볍게.
- ❌ 모든 항목 사전 AI 채점 — 비싸다. AI는 **볼 때만** 호출.
- ❌ skill 내용 자동 수정/병합 실행 — 병합은 **제안**까지, 적용은 사람이 확인.
