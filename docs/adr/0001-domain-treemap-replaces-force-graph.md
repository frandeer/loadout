# 관계 그래프(force-directed)를 도메인 군집 트리맵으로 대체

## Status

accepted

## Decision

기존 "그래프" 탭(React Flow + d3-force node-link)을 삭제하고, 그 자리에 **도메인 군집 트리맵
"무기고 지도"**를 둔다. 트리맵은 `category`별 블록(면적 = 상시 컨텍스트 토큰 무게 추정),
블록 안에 자산 타일(색 = 로드 상태: 장착/설치 베이스/보관)로 구성된다. 대시보드는 별개로
*요약 다이제스트*(총 무게 + 정리 후보 + 최근 변경)를 유지한다 — 지도는 *탐색 캔버스*, 대시보드는
*3초 스캔*이라는 다른 줌 레벨의 역할 분리.

## Considered Options

- **node-link 관계 그래프 유지/개선** — 엣지가 `동명/변형`(중복 신호)과 `태그 교집합 유사도`
  둘뿐이라, 둘 다 *사용 맥락의 관계*가 아니다(전자는 dedup, 후자는 막연한 헤어볼). 코드 주석조차
  "헤어볼 방지"와 싸우고 있었다. 정작 갖고 싶은 `동반(co-use)` 관계는 사용 로그가 희소해 정직하게
  그릴 수 없다.
- **도메인 군집 트리맵(채택)** — 116개가 12개 category에 ~10개씩 고르게 퍼져 있어(이미 깨끗한
  데이터) 묶음 지도가 가장 잘 맞는다. `대체/변형` 관계는 같은 블록 내 인접 + "변형" 배지로 보존하고,
  무의미한 유사도 엣지는 폐기한다. 면적을 토큰 무게로 두어 착지 의도 A("무엇이 켜졌나/얼마나
  먹나/뭘 꺼야 하나")와 융합한다.

## Consequences

- React Flow / d3-force / `lib/graph.ts`의 엣지 빌더(`SHARES_TRAIT` 등)와 GraphView의 호버·포커스·
  ego 스코프 로직 상당 부분이 제거 대상이 된다(되돌리려면 재구축 비용).
- `co-use(동반)` 관계는 의도적으로 *그리지 않는다* — 미래에 사용 로그가 충분해지면 별도 ADR로
  재검토.
